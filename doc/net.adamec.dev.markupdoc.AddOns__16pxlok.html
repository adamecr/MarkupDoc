Output add-on interface. Add-on is a functionality extending the output generator at defined extension points. Implementing class must have a constructor with single OutputOptions parameter.

public interface IAddOn

Implemented by: net.adamec.dev.markupdoc.AddOns.SourceOnlyPackages.SourceOnlyPackagesAddOn

Remarks

One instance per application run is created at GetAddOns() when a master switch in configuration Output.EnableAddOns is on (false by default) Add-on:

  • Can extend the code model root data - when PriorityRootData > 0
  • Can extend the index page - when PriorityIndexPage > 0
  • Can generate own pages - when PriorityOwnPages > 0
  • Can extend "standard" pages' header - when PriorityPageHeader > 0
  • Can extend "standard" pages' body - when PriorityPageBody > 0
  • Can extend "standard" pages' footer - when PriorityPageFooter > 0
The add-ons are applied in the order given by priority defined for each extension point within the add-on.
