Rendered content of <exception> element of XML Documentation Comments. Important: it returns description list item, so must be encapsulated on higher level !!!
Overrides: ModelElement.RenderElement(IMarkupProvider, Member, bool)

Remarks

Builds the link with the reference according to the cref attribute value. The value represents the Documentation ID that can be matched to the information got during the code analysis or to build the link to MS API reference for the "MS objects". First, it tries to resolve the Member in code being documented (internal link). If not successful, tries to look for the cref (Documentation ID) in the MS API documentation if allowed/available (see MsApiDocEngine for details)

The link text is the the name of linked entity when using internal Link(string, Member) or the cref without the leading type information when rendering . When the link is not constructed ("target unknown"), the cref value without the leading type information is rendered in italic

The <exception> element is then rendered as DescriptionListItem(string, string) where Term is the link and Description is the fully trimmed element content. Important: it returns description list item, so must be encapsulated on higher level !!!

Rendered list element of XML Documentation Comments
Overrides: ModelElement.RenderElement(IMarkupProvider, Member, bool)

Remarks

Gets content of the all child ListItem elements and renders them into the list of required ListType .

  • Bullet list - Each item contains "Term" in bold and "Description" (separated by dash). Both parts are optional. This includes the list header item if present, however, there is no special handling for such item.
  • Numbered list - Each item contains "Term" in bold and "Description" (separated by dash). Both parts are optional. This includes the list header item if present, however, there is no special handling for such item.
  • Table - Rendered as two column table (term and description). Both parts are optional. Header item should be provided to define the table header ("Term" and "Description" are used as defaults)
The rendering uses List<TItemType>(Func<TItemType,string>, IEnumerable<TItemType>, bool) to render the lists and TableHeader(string[]) , TableCols(string[]) and TableFooter() for table rendering. Pseudo-tags TagKeepLineBreak and TagKeepSpace are used in the rendering output, to preserve the whitespaces when the parent element is to be rendered (and its content trimmed)

Remarks

Builds the link with the reference according to the cref attribute value. The value represents the Documentation ID that can be matched to the information got during the code analysis or to build the link to MS API reference for the "MS objects". First, it tries to resolve the Member in code being documented (internal link). The link text is the rendered trimmed content of the tag if available, the name of linked entity when using internal Link(string, Member) or the cref without the leading type information when rendering ExternalLink(string, string) . When the link is not constructed ("target unknown"), the cref value without the leading type information is rendered in italic
